# Екзамен: Міні‑проєкт у Jupyter Notebook (90 хв)

Ця тека містить матеріали для проведення іспиту з курсу "Основи генеративного ШІ" (2 курс, Комп'ютерна інженерія).

## Для студентів
- **Основний шаблон для виконання**: `exam_template.ipynb`
  - Заповніть секцію `[0]` (ПІБ, група, варіант, модель, endpoint).
  - Виконайте послідовно секції `[1]` → `[2]` → `[2b]` (чат-режим) → `[3]` → `[4]` (ваш варіант) → `[5]`.
  - У секції `[2b]` чат-режим вимкнено за замовчуванням. Щоб запустити, встановіть `RUN_CHAT = True`. Для безпеки є ліміт `MAX_TURNS`.
  - Ключі — тільки через `.env` (`GITHUB_TOKEN`). Не друкуйте значення ключів у вивід.
- **Індивідуальні варіанти**: Детальний опис у файлі `Іспит_мініпроєкт_GenAI_осінь_2025.md`
  - **Блок A (1-10)**: Основи комп'ютерних систем
  - **Блок B (11-20)**: Програмне забезпечення та мережі  
  - **Блок C (21-30)**: Інтернет речей та embedded системи
- **Стандартна структура кожного варіанту**: генеруйте 3+ приклади → валідуйте формат → порівняйте результати → зробіть висновки.
- **Іменування файлу перед здачею**: `Прізвище І.Б._varNN.ipynb` (приклад: `Петренко І.О._var07.ipynb`).
- **Завантаження**: в Moodle (Завдання: Іспит: Міні‑проєкт із використанням GenAI (Jupyter Notebook)).

- **Опис іспиту**: `Іспит_мініпроєкт_GenAI_осінь_2025.md`
  - Детальні інструкції для кожного етапу
  - Критерії оцінювання та система балів
  - Кореляція з практичними роботами (ПР2-ПР7)

## Зміст теки
- `exam_template.ipynb` — шаблон для студентів з детальними інструкціями та прикладами
- `Іспит_мініпроєкт_GenAI_осінь_2025.md` — офіційне формулювання завдання, структура, критерії оцінювання

## Вимоги до інструментів
- Endpoint за замовчуванням: `https://models.inference.ai.azure.com`
- Моделі та SDK — ті, що використані у практичних (`azure-ai-inference`, `ChatCompletionsClient`, `jsonschema`, `pandas`)
- Використовуйте `.env` для `GITHUB_TOKEN`. Заборонено хардкодити ключі.

## Структура іспиту (90 хв)
- **[0]** Титул і варіант (5 хв)
- **[1]** Підготовка середовища (5-10 хв)
- **[2]** Базова генерація (15-20 хв)
- **[2b]** Діалоговий режим чат-бота (10-15 хв)
- **[3]** Покращення промпту (20-25 хв)
- **[4]** Індивідуальна частина (25-30 хв)
- **[5]** Рефлексія (5-10 хв)

## Ключові оновлення шаблону
- Детальні покрокові інструкції для кожної секції
- Приклади коду для JSON валідації через `jsonschema`
- Автоматичний аналіз результатів через `pandas`
- Кореляція з практичними роботами (ПР2-ПР7)
- Чіткі критерії оцінювання та система балів
- Компактний та легкий для адаптації код

## Примітки
- Якщо чат‑режим у Jupyter не приймає введення, переконайтесь, що `RUN_CHAT = True` і середовище підтримує stdin.
- За відсутності мережі продемонструйте сформовані промпти, параметри та очікувані формати відповідей.
- Всі варіанти мають однаковий рівень складності та однакову структуру виконання.
